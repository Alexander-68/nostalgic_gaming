IN-PLAY UI KIT

One canvas-drawing language for the surfaces every game shares — buttons, HUD readouts, status LEDs and the start / win / pause / over overlays. Load ng.js then ui.js, and draw with NG.ui.

A Buttons NG.ui.button

Rounded, hairline-bordered, uppercase. Three states: default, active (brighter border + phosphor bloom, for an armed control), and disabled (dimmed). Pass a colour to accent one — amber reads as a data / bonus action.

B HUD readouts NG.ui.chip · NG.ui.led

A muted label and an accented value. Green values are game state; amber is live data (score, best). The status LED breathes when lit — use it for whose turn it is, lives, or an armed digit.

C Overlays NG.ui.overlay

The one modal every game reuses. A corner-bracket reticle frames the panel; the title carries a bloom, one amber line shows live data, and the prompt ends in the blinking block cursor carried from the catalogue. Tone colours the title: win, pause, lose.

D Every ratio sizes derive from the shorter side

The same overlay call, unchanged, across the three design ratios — it reads from the shorter side, so it never needs per-orientation tuning.
